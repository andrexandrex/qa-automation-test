Conclusiones
============

El proyecto cumple con los objetivos principales del reto:

1. Se implemento una base funcional de pytest con Selenium, configuracion
   centralizada, fixture de navegador y capturas automaticas en fallos.

2. Se aplico Page Object Model para separar la logica de interaccion con la UI
   de la logica de prueba. Esto hace que los tests sean mas legibles y que los
   cambios de selectores se concentren en las clases de pagina.

3. El flujo E2E de compra en SauceDemo valida el camino critico del usuario:
   login, seleccion de producto, carrito, checkout y confirmacion final con el
   mensaje "THANK YOU FOR YOUR ORDER!".

4. Las pruebas API de PetStore validan el ciclo CRUD de usuario de forma
   independiente a las pruebas E2E. Se usan datos unicos para evitar colisiones
   y reintentos para manejar posibles demoras de consistencia del servicio.

5. El proyecto genera reportes HTML y guarda capturas en fallos E2E, lo que
   facilita revisar resultados y depurar problemas.

6. Se agregaron workflows de GitHub Actions para ejecutar E2E y API en CI. Son
   opcionales para correr localmente, pero aportan valor al proyecto porque
   permiten validar cambios automaticamente al hacer push o pull request.


Hallazgos importantes
---------------------

- SauceDemo puede mostrar alertas de Chrome por la password publica
  secret_sauce. No se considero un defecto del flujo porque forma parte de las
  credenciales demo del sitio.

- Durante la automatizacion, el campo postal code de SauceDemo fue inestable en
  algunos intentos de escritura con Selenium. Se resolvio verificando el valor
  ingresado y usando un fallback que notifica correctamente el estado de React.

- Los botones del checkout estan cerca de una zona de footer fija, lo que puede
  hacer que algunos clicks por coordenadas sean fragiles. Para esas acciones
  especificas se uso un click por JavaScript dentro del Page Object.

- PetStore es un servicio publico de demostracion. Puede responder lento o con
  pequenas demoras de disponibilidad de datos, por eso las verificaciones del
  CRUD incluyen reintentos.


Resultado final
---------------

El proyecto queda listo para ejecutarse desde cero con:

    python -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    pytest

El reporte final se genera en:

    reports/report.html
